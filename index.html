using CSS pseudo selectors to detect click events

:target

  • Set styles for when an element linked to an anchor tag is clicked
  • Can cause the page to jump
  • Only works once

using CSS pseudo selectors to detect click events

:focus

  • Relies on the user focusing on an element
  • Only works with elements that are focusable
  • Clicking anywhere else reverts to the default styling

using CSS pseudo selectors to detect click events

:active

  • Relies on insane cleverness
  • Store an infinite transition on an element
  • On click of the element, override the infinite transition with an instant one
  • On release, the infinite transition preserves any styles applied while :active

using CSS pseudo selectors to detect click events

:checked

  • Works with radio and checkbox inputs
  • The most flexible of the lot, IMHO
  • Allows us to work with a boolean, :checked or unchecked

CSS


/* A BUNCH OF STYLES THAT AREN'T TOO IMPORTANT */

label[for*="widget"]:hover ~ .question-group .question-answer {
  will-change: contents;
}

/*SET UP DEFAULT STYLE FOR TOGGLE-ALL GROUP AND INDIVIDUAL WIDGET*/
.details.hidden-checkbox ~ .question-answer {
  max-height: 0;
  padding-bottom: 0;
  transition: all 400ms ease-in-out;
}

/*TOGGLE ALL WIDGETS OPEN/CLOSED AND TOGGLE INDIVIDUAL WIDGET OPEN/CLOSED*/
.details.hidden-checkbox:checked ~ .question-answer,
.toggle-all.hidden-checkbox:checked ~ .question-group .question-answer {
  padding-bottom: 15px;
  max-height: 1000px;
}

/*TOGGLE INDIVIDUAL WIDGETS OPEN/CLOSED WHILE TOGGLE-ALL IS :CHECKED*/
.toggle-all.hidden-checkbox:checked ~ .question-group .details.hidden-checkbox:checked ~ .question-answer {
  max-height: 0;
  padding-bottom: 0;
}

/*ROTATE CHEVRONS WHEN ALL WIDGETS ARE TOGGLED*/
.toggle-all.hidden-checkbox:checked ~ .question-group label .chevron,
.toggle-all.hidden-checkbox ~ .question-group .details.hidden-checkbox:checked + label .chevron {
  transform: rotate(180deg);
}

/*ROTATE INDIVIDUAL CHEVRONS WHILE TOGGLE-ALL IS :CHECKED*/
.toggle-all.hidden-checkbox + label .chevron,
.toggle-all.hidden-checkbox ~ .question-group label .chevron,
.toggle-all.hidden-checkbox:checked ~ .question-group .details.hidden-checkbox:checked + label .chevron {
  transform: rotate(0deg);
}

Pros? Whaddaya think?

Totally Impartial...

PROS

  • Wide support: good to go from IE9 and up!
  • Only a small amount of extra markup needed to make it work
  • No javascript required! Hurrah!
  • It will work no matter what, totes progressive enhance
  • This is CSS badassery at its finest

Cons? Whaddaya think?

Totally Impartial...

CONS

  • :checked does not have deepest browser support
  • Extra markup needed to make it work
  • Using CSS to do javaScripts job is bad, mkay
  • Not accessible
  • CSS selectors can get overly complex and unreadable
